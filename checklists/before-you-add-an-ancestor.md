# Before You Add An Ancestor

Use this before adding a parent, spouse, sibling, or child to the tree.

## Minimum Evidence

- [ ] You have opened the source yourself.
- [ ] The source connects the person to someone already in your tree.
- [ ] The date range is biologically and historically plausible.
- [ ] The location fits known migration or residence patterns.
- [ ] The relationship is stated or strongly supported, not only inferred from a shared surname.

## Blockers

- [ ] Do not add an ancestor from one user-contributed tree alone.
- [ ] Do not add a parent only because the names match.
- [ ] Do not merge two people because they share a birth year.
- [ ] Do not treat Find a Grave, an index, or a copied GEDCOM as final proof by itself.
- [ ] Do not add a living person without privacy review.

## After Adding

- [ ] Add the source citation.
- [ ] Set `evidence_tier` to `strong_signal`, `moderate_signal`, or `speculative`.
- [ ] Add unresolved issues to `Open_Questions.md`.
- [ ] Add searches that failed to `Research_Log.md`.
- [ ] Run [02 Cross-Reference Audit](../prompts/02-cross-reference-audit.md) after a major tree change.

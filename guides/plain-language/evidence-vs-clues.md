# Evidence Vs Clues

Genealogy moves faster when clues and evidence stay separate.

## The Difference

| Type | What it does | How to use it |
|---|---|---|
| Evidence | Supports a claim about a person, date, place, or relationship | Add it to the tree with a citation |
| Clue | Suggests where to search next | Put it in `Open_Questions.md` or `Research_Log.md` |

## Evidence

Evidence answers a specific question.

Fictional example:

A baptism record names Taylor Finch, gives a baptism date, names the parents, and is from the town where the family lived. That is evidence for parentage.

## Clue

A clue points toward a possible answer.

Fictional example:

A DNA match's public tree includes the Finch surname in the same county. That does not prove a relationship. It suggests a search direction.

## Common Clues People Mistake For Evidence

- DNA ethnicity percentages.
- Small chromosome segments.
- User-contributed trees.
- Search-result snippets.
- Photo captions.
- Family stories.
- Index entries with no image.

## How To Record Each One

For evidence:

- Add the citation.
- Add the fact.
- Set the evidence tier.
- Note any conflict.

For clues:

- Add the lead to `Open_Questions.md`.
- Add the search plan.
- Keep the tree unchanged until a source supports it.

## DNA Reminder

DNA can be excellent for finding leads. DNA rarely proves a specific ancestor by itself. A DNA lead becomes stronger only when records support the same relationship.

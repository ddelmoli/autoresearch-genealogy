# Vault File Manifest

Canonical file names for the genealogy vault template. Prompts and workflows should use these names exactly to avoid duplicate files on case-sensitive file systems.

## Core Files

| Purpose | Canonical file |
|---|---|
| Master index | `_Index.md` |
| Family tree | `Family_Tree.md` |
| Research log | `Research_Log.md` |
| Open questions | `Open_Questions.md` |
| Data inventory | `Data_Inventory.md` |
| Timeline | `Timeline.md` |
| Genetic profile | `Genetic_Profile.md` |
| Chromosome painting | `Chromosome_Painting.md` |
| Witness network | `Witness_Network.md` |
| Unresolved people | `Unresolved_Persons.md` |
| Research strategy | `Research_Strategy.md` |

## Generated Audit Files

These files are generated by prompts and are intentionally lowercase because they are machine-oriented audit outputs.

| Purpose | Canonical file |
|---|---|
| Cross-reference audit | `cross_reference_audit.md` |
| Find a Grave audit | `findagrave_audit.md` |
| GEDCOM audit | `gedcom_audit.md` |
| Source citation audit | `source_citation_audit.md` |
| Timeline gap tracker | `timeline_gaps.md` |

## Privacy Fields

Person files should include:

```yaml
life_status: living | deceased | unknown
evidence_tier: strong_signal | moderate_signal | speculative
profile_status: complete | partial | stub
```

Autonomous prompts must skip people with `life_status: living` or `life_status: unknown` unless the task is explicitly limited to local, private vault cleanup.
